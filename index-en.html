Mes projets

ProxySwitcher

I like developing scripts to simplify my life. For example, in high school, Linux connections were consistently denied. Two friends and I investigated the issue and found that the proxy was rejecting the connections. So, we analyzed the traffic with Wireshark and found the IP address and port of the proxy. To avoid having to reconfigure the proxy server every time I needed to switch between my personal network and the school's, I developed this tool to do it for me.

The script prompts to enter the IP address and PORT of the proxy during the first connection, but it's possible to change this setting anytime. The script then saves this data in a file named optionPS.txt for reuse whenever we want to reactivate the proxy.

Serveur de stockage de fichiers et de gestion de torrents

Storing files takes up space. So, I often have to store files on an external hard drive for backup. However, this has a drawback: if we don't have the external hard drive with us when we need it, the data becomes inaccessible. Also, cloud solutions are often paid if you want to store a large amount of data. So, I decided to create my own file storage server. For this, I retrieved and repaired a defective PC from my high school. I added a 2TB hard drive in addition to the existing one:

I installed Ubuntu on it, assigned it a static IP address, and can now connect to it via SSH. From SSH, I configured my server to add a torrent manager to it, so I can easily send my files. So, I chose qBittorrent because it has a web interface:

You just need to specify the IP of the web server (here my server) to the program and open the port on my router to access it from anywhere. We don't forget to add a strong password, of course. To easily start my torrent manager on startup, I add a service that we don't forget to activate, and it's ready!
