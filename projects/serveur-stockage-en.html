My Projects: File Storage management Server

Context:

Storing files can become cumbersome, especially in terms of space and accessibility. External hard drives are often used for backups, but they require physical presence. Additionally, cloud storage solutions are often costly for large amounts of data. To solve this problem, I decided to create my own file storage server.

Objective:

The main objective was to have a storage space accessible from anywhere, without relying on physical media or paid services. To achieve this, I recovered a faulty PC, repaired it, and equipped it with a 14 TB hard drive. By installing Debian and configuring a static IP address, I made server access easier via SSH. Then, I installed VSFTPd, which allows for easy and more secure file uploads and downloads compared to a simple FTP server. The key is to carefully manage user permissions and access rights.

I also had to address the issue of external access to the network. Since I live in a student residence, I do not have access to the router configuration and therefore cannot port forward the various services on my server. To work around this limitation, I used SSH to create a tunnel between my local server and my cloud-based VPS, making it accessible from outside the network.

Skills Developed:

Hardware Repair and Configuration: Recovering and repairing a defective PC, adding an extra hard drive.

System Administration: Installing and configuring Debian, assigning a static IP address.

Network Management: Usage of SSH tunnels to allow remote access.

Software Usage: Installing and configuring vsFTPd for files management.

Cybersecurity: Adding IP rules to allow only known IPs and securing server access with a strong password.
