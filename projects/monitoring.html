Mes projets : Interface web de monitoring

Contexte :

Dans le cadre de la gestion d'un serveur personnel ou professionnel, il est essentiel de pouvoir surveiller les performances, l'état des services, et la connectivité. Un site de monitoring centralisé permet à un administrateur de vérifier l'état du serveur en temps réel, d'effectuer des contrôles à distance et d'optimiser la gestion des ressources.

Objectif :

Développer un site web de monitoring pour surveiller les différents aspects d'un serveur, tels que les performances, le réseau, le stockage, les services en cours d'exécution, et les contrôles de serveur. Le but est de permettre une gestion facile et une supervision efficace du serveur grâce à une interface utilisateur claire et des options de contrôle dynamiques.

Détails :

Fonctionalités principales

Le site web possède plusieurs catégories ( Général, Performances, Réseau, Stockage, Services, Contrôle ). Ces catégories possèdent chacune leurs propres fonctionnalités pour gérer différents aspects et afficher des informations en fonction de certains événements. Par exemple, la partie “Performance” affiche des informations comme l’utilisation de la ram ou du CPU à l'aide de jauges pour plus de clarté. La page “Réseau”, elle, affiche les informations de chaque carte réseau ainsi que les informations associées par exemple eno1 : 20ms, IP: 192.168.1.50 Masque : 255.255.255.0 DNS : 8.8.8.8 Active. La catégorie “Stockage” affiche une jauge ainsi que le pourcentage d’utilisation de chaque disque installé en Go et la catégorie “Services” rend compte des différents services actifs sur le serveur et leur statut, par exemple Apache 2 : actif, Apache 2 : en erreur, inactif, etc. Enfin, la partie “Contrôle” gère l'arrêt, le démarrage ou le contrôle vpn du serveur via une interface web

Technologies utilisées

HTML/CSS : pour la structure et le style de l'interface.

JavaScript : pour les interactions utilisateur et les modifications dynamiques (ex. : gestion du VPN, mise à jour de l'état des liens de navigation actifs).

PHP : pour gérer la logique côté serveur ( ex. : arrêt/redémarrage du serveur, activation du vpn coté serveur).


Compétences développées :

Développement Front-End : Conception d'une interface utilisateur réactive et attrayante en utilisant HTML, CSS et JavaScript.

Gestion de serveur web : Configuration de la navigation, gestion des états des services et développement de contrôles administratifs (ex. : redémarrage de serveur, suivi du stockage).

Programmation PHP Création de scripts pour la gestion des éléments interactifs, comme l'activation du VPN.

Structuration du CSS : Organisation et gestion des styles CSS pour maintenir une apparence cohérente et ergonomique à travers toutes les pages.

Approche modulaire : Développement d'une interface web qui peut être étendue ou modifiée facilement pour ajouter de nouvelles fonctionnalités.
