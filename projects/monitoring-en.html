My Projects: Web Monitoring Interface

Context:

In managing a personal or professional server, it is essential to monitor performance, service status, and connectivity. A centralized monitoring website allows an administrator to check the server's status in real-time, perform remote checks, and optimize resource management.

Objective:

To develop a monitoring website for overseeing various server aspects, such as performance, network, storage, running services, and server controls. The goal is to enable easy management and effective supervision through a clear user interface and dynamic control options.

Details:

Main Features

The website includes several categories (General, Performance, Network, Storage, Services, Control). Each category has its own functionalities to manage and display information based on specific events. For example, the "Performance" section displays information such as RAM and CPU usage using gauges for clarity. The "Network" section shows details for each network interface, such as eno1: 20ms, IP: 192.168.1.50, Subnet Mask: 255.255.255.0, DNS: 8.8.8.8, Active. The "Storage" category displays a gauge and the percentage of disk usage in GB, and the "Services" section reports on active services and their statuses, e.g., Apache 2: active, Apache 2: error, inactive, etc. Lastly, the "Control" section handles server shutdown, startup, or VPN control via a web interface.

Technologies Used

HTML/CSS: For structuring and styling the interface.

JavaScript: For user interactions and dynamic updates (e.g., VPN management, updating active navigation links).

PHP: For server-side logic (e.g., shutting down/restarting the server, enabling server-side VPN).


Skills Developed:

Front-End Development: Designing a responsive and attractive user interface using HTML, CSS, and JavaScript.

Web Server Management: Configuring navigation, managing service states, and developing administrative controls (e.g., server restarts, storage tracking).

PHP Programming: Writing scripts for managing interactive elements, such as enabling the VPN.

CSS Structuring: Organizing and managing CSS styles to maintain a consistent and ergonomic appearance across all pages.

Modular Approach: Developing a web interface that can be easily extended or modified to add new features.
