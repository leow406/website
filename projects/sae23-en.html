My Projects: Ticketing Solution

Context:

In the context of an SAE, the university faced a need for efficient management of IT support requests within its IT Department (DSI - Direction des Systèmes d'Information). To address this requirement, a project was initiated to develop a web-based ticketing management system. This website would allow users to report issues, track their status, and resolve them transparently. The system administrator could manage tickets based on their priority and impact.

Objective:

The main goal of this project is to design and deploy a full-stack ticket management application using Node.js and MongoDB as the database. The application must provide multiple features such as ticket creation, tracking, and deletion by users, management and classification of tickets by administrators, the ability to assign technicians, and converting private tickets to public tickets to help other users facing similar issues. The system must be robust, scalable, and user-friendly to ensure smooth handling of requests.

Skills Developed:

Full-Stack Programming: Using Node.js for server-side logic and MongoDB for data management.

User and Role Management: Implementing a ticketing system with different access levels (users, administrators, technicians).

Web Architecture: Developing a complete architecture for both server and client sides.

Technologies: Familiarizing with tools such as Express.js, MongoDB, and Node.js to build REST APIs and real-time applications.
