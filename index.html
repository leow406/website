Mes projets

ProxySwitcher

Contexte :

Au lycée, l'accès aux connexions Linux était régulièrement bloqué, posant des difficultés pour travailler efficacement. Pour résoudre ce problème, j'ai entrepris de développer un script permettant de contourner les restrictions du proxy qui rejetaient les connexions.

Objectif :

L'objectif principal était de simplifier le processus de basculement entre mon réseau personnel et celui du lycée en évitant de devoir reconfigurer manuellement le proxy à chaque fois. Le script devait permettre de saisir l'adresse IP et le port du proxy lors de la première connexion, tout en offrant la flexibilité de modifier ces paramètres à tout moment. De plus, les données devaient être enregistrées dans un fichier pour une utilisation ultérieure.

Compétences développées :

Programmation : Développement d'un script pour gérer les paramètres de proxy.

Analyse réseau : Utilisation de Wireshark pour identifier l'adresse IP et le port du proxy.

Gestion de fichiers : Enregistrement et récupération des données dans un fichier texte depuis un script.

Serveur de stockage de fichiers et de gestion de torrents

Contexte :

Stocker des fichiers peut devenir contraignant, notamment en termes d'espace et d'accessibilité. Les disques durs externes sont souvent utilisés pour les sauvegardes, mais ils nécessitent d'être physiquement présents. De plus, les solutions de stockage cloud sont souvent payantes pour de grandes quantités de données. Pour résoudre ce problème, j'ai entrepris de créer mon propre serveur de stockage de fichiers.

Objectif :

L'objectif principal était de disposer d'un espace de stockage accessible de n'importe où, sans dépendre d'un support physique ou de services payants. Pour cela, j'ai récupéré un PC défectueux que j'ai réparé et équipé d'un disque dur de 2 To. En installant Ubuntu et en configurant une adresse IP statique, j'ai facilité l'accès au serveur via SSH. Ensuite, j'ai ajouté un gestionnaire de torrents, en l'occurrence qBittorrent, pour simplifier l'envoi de fichiers. La configuration de l'interface web du gestionnaire de torrents permet un accès facile à distance, avec la sécurisation par un mot de passe robuste.

Compétences développées :

Réparation et configuration matérielle : Récupération et réparation d'un PC défectueux, ajout d'un disque dur supplémentaire.

Administration système : Installation et configuration d'Ubuntu, attribution d'une adresse IP statique.

Gestion de réseau : Ouverture des ports sur la box pour permettre l'accès à distance.

Utilisation de logiciels : Installation et configuration de qBittorrent pour la gestion des torrents.

Sécurité informatique : Ajout de regles IP pour n'autoriser que les IP connues et ajout d'un mot de passe fort pour sécuriser l'accès au serveur.
